Product Management System Documentation

Introduction

Welcome to the documentation for the Product Management System project. This project is designed to showcase the implementation of a basic Product Management System using ASP.NET WEB API, Blazor front-end and C#. The system encompasses essential modules such as Item Management, Customer Management, Customer-Item Relationship Management, and Reporting functionalities.

The primary objective of the application is to empower employees with the ability to efficiently manage customers, items, and their associations, while also facilitating the generation of insightful reports. The system operates with a Sqlite database backend to ensure persistent storage of data.

Key Features:

  • Clean Front-End: The user interface is designed to be intuitive and user-friendly, ensuring a smooth user experience.
  • Blazor Front-End Framework: The project utilizes the Blazor framework for the front-end, leveraging its capabilities for building interactive web applications with C#.
  • Form Validations: Both backend and frontend form validations are integrated to enhance data integrity and user input validation.
  • Documentation: Comprehensive documentation is provided to aid in understanding the project structure, implementation details, and usage instructions.
  • Custom Reports: A custom report feature is implemented to generate reports showcasing customers along with their X most expensive items.
  • Login Requirement: Security is enforced through a login requirement using ASP.NET Identity, ensuring that only authorized users can access the system.

Structure

The project is structured following the principles of Clean Architecture, which emphasizes separation of concerns and maintainability. The Clean Architecture approach ensures that the system's core business logic remains decoupled from external dependencies, facilitating easier testing, scalability, and future modifications.

The project consists of the following layers:

  • Domain Layer:
    • This layer contains the core business logic and domain entities. It encapsulates the fundamental rules and concepts of the application, independent of any specific technology or framework. Entities, value objects, and domain services reside in this layer.
  • Infrastructure Layer:
    • The Infrastructure layer houses external concerns such as database access, file I/O, third-party integrations, and other infrastructure-related components. Repositories, database contexts, and external service clients are typically implemented in this layer.
  • Application Layer:
    • The Application layer serves as an intermediary between the Domain and Infrastructure layers. It orchestrates interactions between domain entities and infrastructure components, implementing use cases and application-specific business logic. This layer often contains application services, use case interactors, and other application-specific logic.
  • Client Blazor UI:
    • The Client Blazor UI layer represents the user interface of the application, built using the Blazor framework. It consumes the API exposed by the Application layer to interact with the system. Components, pages, and client-side logic are implemented in this layer to provide an interactive and responsive user experience.
